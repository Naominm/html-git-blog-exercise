2. What is Git

Git is a distributed version control system used to track changes in files and coordinate work among developers.

What is a Version Control System (VCS)?

A VCS is a tool that allows developers to:

  • Track changes made to files.
  • Revert to previous versions if needed.
  • Collaborate seamlessly on projects.

other version control system include:

  • Subversion (SVN)
  • Mercurial
  • Perforce

Installing Git

If Git is not installed, follow these steps:

  1. Go to Git’s official website.

  2. Download and install the appropriate version for your operating system (Windows, macOS, or Linux).

6. Adding files to the staging area

What is the Staging Area?

The staging area is a space where Git prepares changes before committing them.

steps to add files

  1. To stage a single file:

    2. git add filename

  2. To stage all files

    3. git add .

Tracked vs Untracked files

  • Tracked files: Files already added to Git's tracking system.

  • Untracked files: New files not yet added to Git.

7. Commiting changes

What is a commit?

A commit is a snapshot of your project at a particular time.

Steps to commit

Git commit -m "Your Message"

Commit changes with a message

Viewing Past Commits

git log

Reverting to a Previous Commit

git checkout commit-hash

8. Branches

Branches allows you to work on different versiopns of your project simutaneously. For example you can develop a new feature on a new branch without affecting the main branch

Creating Branches

create a branch using "git branch branch-name"

Viewing all Branches

view all branches with:

git branch

  • The active branch is marked with an asterisk (*).

Switching Active Branches

Switch between branches using:

git switch branch-name

Merging a Branch

Merge a branch into the active branch using:

git merge branch-name

Deleting a branch

Delete a branch you no longer need using:

git branch -d branch-name
